# Practical Workbook — Music of Objects × Modality × Juggling

Daily reference for scene work, notation gaps, attention, and fixation.  
Sources: [Music of Objects](../20260623120000_music_of_objects.md) · [Modality of Spinning](../20260612120000_o_modalnosti_krucheniya.md) · [Juggling](../20260427101600_juggling_en.md)

Blank forms: `template_scene.txt` · `template_exercise.txt` · `template_fixation.txt` · `template_materialization.txt` · `template_project_documentation.txt`

---

## How to use this

- Pick **one scene** (5–15 min), not a program.
- Speak **one sentence** of intention before acting.
- After: **fixate** in two minutes — name difference and error, not beauty.
- Constraints **expire** when the situation changes (shoulder, room, partner, fatigue).
- Copy a template file, rename by date, fill in, archive.

**Working line**

`intention → action → difference → error → awareness → repetition → variation → scene`

**Break line (text / code / notation)**

`break → observation → check → correction → habit named`

---

## Shared ideas (all three sources)

- Object work is **relation**, not repertoire alone.
- **Attention** is the primary instrument when hands cannot hold all variants.
- **Inner state** shows in outer form (clenched vs open; automatism vs intention).
- **Rhythm** is a grid (siteswap, count, voice, sync mode) laid over continuous body action.
- **Continuous recovery**: fall, catch, redistribute — poi, voice, code, daily gesture.
- **Formal agreement ≠ same action**: same pattern, different meaning between bodies or props.
- **Slippage is material**: unnamed error stays fog; named error becomes vocabulary.
- **Situation first**: this shoulder, this cable, this friendship, this weather — no rule above that.

## Daily workflow

### 15 minutes

- [ ] Choose intention (one sentence)
- [ ] Speak the scene aloud
- [ ] Perform slowly
- [ ] Increase speed (50 → 70 → 90 → 110 → return to 60)
- [ ] Repeat one error on purpose
- [ ] Fixate (short notes)

### 45 minutes

- [ ] Two scenes: object + sound (or silence)
- [ ] One voice or score segment
- [ ] One anti-instruction pass
- [ ] Fixate each scene
- [ ] One line: what to try next time

### 90 minutes

- [ ] Movement without sound
- [ ] Sound without movement
- [ ] Relation study (one sync mode, deep)
- [ ] Voice or written score
- [ ] Partner play OR notation/score reading OR text/code pass
- [ ] Recorded form OR materialization sketch
- [ ] Project log entry (one line)

### Any day, no props

- [ ] Micro-juggling: pass, hold, or transfer one everyday object three different ways
- [ ] Chair scene: no object, room sound only, 3 min
- [ ] Say intention → do → say matched / did not match / what differed

Use `template_scene.txt`. Minimum fields:

- [ ] **Name** (short, memorable)
- [ ] **Situation** (body, room, company, fatigue)
- [ ] **Object** (poi, ball, staff, hand, cursor, code, voice, none)
- [ ] **Sound condition** (source on, silence, room only, expected sound absent)
- [ ] **Duration**
- [ ] **Intention** (one sentence, spoken)
- [ ] **Action** (10 sec of clear motion beats 5 min of fog)
- [ ] **Freedom bounded** (what you narrowed on purpose)
- [ ] **Held back** (decoration, defaults, tricks you already know)
- [ ] **Sync mode** (pick one; see Sync relation picker below)
- [ ] **Expected difference**
- [ ] **Completion** (how you know the pass is done)

### Notation layer (when using siteswap or score)

Add when pattern or external structure is involved:

- [ ] **Pattern** (e.g. `3`, `5223`, or custom score strip)
- [ ] **Prop modality** (rigid / poi / mixed / none)
- [ ] **Dwell** (short / long / active rotation / omitted — note if notation omits it)
- [ ] **Spatial** (plane, direction, radius — note if notation omits it)
- [ ] **Agreement target** (rhythm only / rhythm + kinematics / rhythm + meaning)

## After the pass — fixation checklist

Use `template_fixation.txt`. Minimum fields:

- [ ] Date, scene name, situation (as lived)
- [ ] Intention (what was said)
- [ ] What was done
- [ ] **Difference** (gap; no “bad” / “wrong”)
- [ ] Error
- [ ] Useful error? (yes/no + why)
- [ ] What speed destroyed
- [ ] Anti-instruction for next pass
- [ ] Repeat / hold back

### Cross-source fixation extras

When mixing notation + body + attention:

- [ ] **Notation used**
- [ ] **Rhythm matched?** (yes/no)
- [ ] **Kinematics matched?** (yes/no — throw vs sustain, plane, dwell)
- [ ] **Meaning matched?** (yes/no — did you “feel” the same pattern?)
- [ ] **Insufficiency** (what parameter would the score need: dwell, plane, direction, acceleration character?)
- [ ] **Inner state** (clenched / open / automatism / return of control)
- [ ] **Siteswap lied?** (notation valid but body under-specified)

## Evaluation (skip beautiful / ugly)

Half true is enough to keep the scene.

- [ ] Intention was explicit
- [ ] Difference could be named
- [ ] Error appeared and could repeat
- [ ] Something changed at higher speed
- [ ] Sound relation had character (if sound was in play)
- [ ] Silence functioned as material
- [ ] Scene worked without active sound source
- [ ] Another person could enter
- [ ] Same scene could move to object, voice, or text

## Exercise rooms

Use `template_exercise.txt`. Pass through modes in order; fixate after.

### One gesture, one sound

- Object: poi, ball, staff, hand, cursor
- Sound: tone, click, burst, or none
- Duration: 3 min
- Modes: synchrony → 0.5s delay → 0.5s prediction → every 3rd only → no sound, attention only
- Ask: clearest relation? character? repeatable error?

### Pause as action

- Circles and stalls only
- Variants: move sounds / pause silent; move silent / pause sounds; both silent, attention shifts
- Expected error: leaving pause too early
- Completion: one pause stronger than movement before it

### Fall and catch

- Materials: noise + tone (or silence as third)
- Variants: fall sounds / catch silent; reverse; sound between; soft catch only; expectation only
- Cross: throw (rigid) vs sustained transfer (poi) — same exercise, different modality

### Phase between two lines

- Lines: hands, poi, hand + voice, ball + poi
- Modes: together → drift → quarter shift → half shift → forced return
- Expected error: one line controls the other
- Completion: hear the relation

### Resistance

- Sequence: mirror → resistance → ignore every 3rd → fatigue weakens response → large gesture → silence reset
- Cross: bigger throw ≠ bigger sound; `5` in siteswap = acceleration, not height

### Error as vocabulary

- Perform familiar pattern without hiding slips
- Assign: phase loss → swing; tremor → tremolo; early catch → prediction; late catch → delay; drop → reset; forgotten pause → syncopation; premature prep → precursor
- Repeat chosen error ×3 on purpose

### Voice as metronome

- Count: “one two three pause” / “circle circle stop silence” / nonsense syllables
- Variants: voice leads body; body leads voice; voice accelerates alone; movement accelerates alone; voice commands silence
- Expected error: voice serves habit, not direction

### Anti-instruction

- On “circle”: stall, silence, other plane — not circle in disguise
- On “louder”: density, distance, grain, delay, dryness — not volume alone
- On siteswap digit: any response except literal interpretation
- On “use if” (code): another shape without if/switch/ternary

### Speed ladder

- 50% → 70% → 90% → 110% → return 60%
- Note what destroyed: command order, dwell, phase, intention, notation agreement

### Micro-juggling (daily, no mat)

- Any object: cup, pen, door handle, blade of grass
- 30 sec: three transfers, three tempos, three paths (below / side / across)
- Fixation: one line only

## Notation & siteswap — practical rules

From Modality of Spinning. Use siteswap as **ordering grid**, not full movement language.

### When plain siteswap is enough

- Rigid props
- Throw → catch primary movement
- Solo practice with agreed convention
- Rhythm-only ensemble check

### When plain siteswap is not enough

- Poi / flexible / torque-based props
- Mixed ensemble (ball + poi same pattern)
- Group sync without agreed plane, dwell, direction, acceleration
- Any pass where “same `3`” feels like different action

### Parameters to add (annotation or fixation)

- **Dwell time** (short / long / third vs half beat — or active rotation phase)
- **Plane** (wall, wheel, mixed)
- **Rotation direction**
- **Acceleration character** (`5` as intensified spin, not throw height)
- **Spatial path** (spiral, weave, pendulum, correction)

### Pattern reminders

- `3` on balls: alternating cascade rhythm
- `3` on poi: wall spiral, weave, pendulum — same rhythm, many forms
- `5223`: in poi, `2` may be active rotation; `5` may be intensified acceleration
- “No cascade in poi” = no throw/catch form transfer; rhythmic analogue may still exist

### Code as materialization (modality folder)

- `modality_of_spinning.feature` — scenarios as readable score
- `modality_of_spinning.rb` / `.ex` — extended model with `dwell_time`, `spatial`
- `InsufficientNotation` — stop and fixate before forcing the pattern
